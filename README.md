# Адель — семейный бюджет

Полноценное приложение для совместного ведения бюджета: бэкенд на Express/Prisma/PostgreSQL, PWA-фронтенд на React (устанавливается на телефон как приложение), отдельная админ-панель, push-уведомления и ИИ-разбор трат.

## Структура репозитория

```
/server   — Express API + Prisma (авторизация, пространства, транзакции, бюджеты, цели, пуши, ИИ-прокси, админ-эндпоинты)
/web      — семейное PWA-приложение (React + Vite)
/admin    — админ-панель для разработчика (React + Vite, отдельный деплой и авторизация)
/shared   — общие TypeScript-типы и Zod-схемы валидации
```

## Локальный запуск

Требуется Node.js 20+ и доступ к PostgreSQL (см. `DEPLOY.md` про бесплатный Neon).

```bash
npm install                     # ставит зависимости всех workspace-ов сразу

cp server/.env.example server/.env   # заполнить DATABASE_URL и секреты
cp web/.env.example web/.env.local
cp admin/.env.example admin/.env.local

cd server && npx prisma migrate dev   # применить схему БД
npm run seed:admin --workspace server # создать первого админа (по ADMIN_SEED_* из .env)

npm run dev:server     # http://localhost:4000
npm run dev:web        # http://localhost:5173 — семейное приложение
npm run dev:admin      # http://localhost:5174 — админ-панель
```

## Что реализовано

- Регистрация/логин по email+паролю, JWT с ротацией refresh-токена
- Пространства (workspace) с ролями владелец/участник, приглашения по ссылке — можно вести бюджет вдвоём, а в будущем добавить больше участников
- Категории, операции, бюджеты по категориям, финансовые цели — с ролевыми проверками на каждом запросе
- ИИ-разбор трат из свободного текста (прокси к Anthropic на сервере, ключ никогда не уходит в браузер)
- Push-уведомления (партнёр добавил операцию, превышен бюджет, еженедельная сводка, новый участник) — веб-пуши, работают в PWA на Android везде, на iOS только после установки на экран «Домой» (ограничение Apple)
- Отдельная админ-панель со своей авторизацией: список пространств/пользователей, отключение/удаление аккаунтов, журнал событий
- Rate-limiting, helmet, CORS allowlist, Zod-валидация всех запросов, workspace-scoped запросы везде (защита от IDOR)
